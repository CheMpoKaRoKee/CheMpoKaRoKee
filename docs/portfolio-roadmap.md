# Portfolio Roadmap

Этот файл фиксирует направление GitHub-портфолио под позиционирование:

```text
AI Automation / API Integrations / Telegram Bots
```

Главная цель — показывать не просто умение писать код, а умение собирать рабочие инструменты из существующих сервисов, API и AI-моделей.

## Текущее позиционирование

Основная роль:

```text
AI-интегратор / автоматизатор
```

Ключевые направления:

- AI automation;
- API-интеграции;
- Telegram-боты;
- Python для автоматизации;
- NocoDB / Google Sheets / Apps Script;
- рабочие ассистенты;
- внутренние инструменты для процессов;
- GitHub-инструменты и документация.

Backend и FastAPI — не главная карьерная рамка, а вспомогательные инструменты для API-сервисов и автоматизаций.

## Следующий сильный проект

### AI Operations Assistant

Идея:

```text
AI-помощник принимает задачу, читает данные из таблицы, GitHub или внешнего API, анализирует их, формирует отчёт и создаёт задачу или уведомление в Telegram/NocoDB.
```

Почему этот проект важен:

- напрямую подтверждает позиционирование AI Automation;
- связывает AI, API, GitHub, Telegram и таблицы;
- показывает не чат, а полноценный workflow;
- подходит под реальные задачи малого бизнеса и операционных команд.

Минимальный MVP:

1. входная задача через Telegram или CLI;
2. чтение данных из одного источника;
3. вызов AI для анализа;
4. формирование короткого отчёта;
5. создание задачи или уведомления;
6. логирование результата.

Возможный стек:

- Python;
- FastAPI или CLI как входная точка;
- Telegram Bot API;
- GitHub API;
- Google Sheets API или NocoDB API;
- OpenRouter API;
- SQLite/PostgreSQL для логов;
- Docker для запуска.

## Что не делать

- Не позиционировать профиль как классический backend, пока главная цель — интеграции и автоматизация.
- Не распылять GitHub на случайные тестовые репозитории.
- Не показывать приватные рабочие названия и внутренние процессы.
- Не добавлять скриншоты с реальными клиентскими данными.
- Не публиковать конфиги с приватными значениями.

## Итоговая рамка

```text
Может ли этот человек быстро собрать полезную автоматизацию из Telegram, таблиц, API и AI-инструментов?
```

Текущие проекты уже начинают отвечать: да. Следующий шаг — добавить визуальные доказательства и один сильный AI Operations Assistant.
